## 1. Xcode 加入 Pace target

1. 打开 `Pace.xcodeproj` (你的本地)
2. Project navigator 左侧 → `Pace` 文件夹右键 → **Add Files to "Pace"…**
3. 多选所有 15 个新 `.swift` 文件 + `Models/` 和 `Services/` 两个文件夹
4. 弹窗选项:
   - ✅ **Copy items if needed** — 不勾(文件已在仓库内)
   - **Added folders**: 选 **Create groups**
   - ✅ **Add to targets: Pace**
5. 点 Add

## 2. Info.plist 加权限说明 (3 条 key)

打开 Project → Pace target → **Info** tab → 在最下面 Custom iOS Target Properties 表里点 `+`:

| Key | Type | Value (中文文案) |
|---|---|---|
| `NSLocationWhenInUseUsageDescription` | String | 跑步时记录路线和配速,仅在跑步中使用。 |
| `NSHealthShareUsageDescription` | String | 用于读取你的心率,在跑步中实时显示。 |
| `NSHealthUpdateUsageDescription` | String | 把这次跑步保存到 Health 应用,方便回看。 |

**没有 NSLocationWhenInUseUsageDescription, CLLocationManager 静默失败 — 拿不到 GPS 也不弹权限框。** 这是新手最常踩的坑。

## 3. 启用 Capabilities (2 项)

Project → Pace target → **Signing & Capabilities** tab → 点左上 `+ Capability`,依次添加:

### 3.1 Background Modes
- 勾 **Location updates** (只勾这一个 — 真跑步要求后台 GPS 不停)

### 3.2 HealthKit
- 勾 HealthKit (这一项会自动写 `com.apple.developer.healthkit` 到 `.entitlements`)
- (`Clinical Health Records` 和 `Background Delivery` 子选项**不勾**)

## 4. 验证编译

```bash
# 选 iPhone 12 Pro 模拟器
Cmd + B 编译
```

预期: build succeeded 0 errors 0 warnings.

如果报 `Cannot find type 'XX' in scope`,通常是某个新文件没加到 target。回到第 1 步确认。

## 5. 验证 GPS 路径

1. 选 iPhone 12 Pro 模拟器 → Run
2. App 启动 → IdleHome → 点"出发"
3. 系统弹**位置权限** dialog → 选 Allow While Using App
4. 系统弹**HealthKit 授权** sheet → 全部允许 (Allow All)
5. 现在屏幕停在 **PreRunView** "搜索 GPS",显示 `0/6 颗`
6. **Mac 顶部 Simulator 菜单 → Features → Location → City Run**
7. 5 秒内 fixCount 应该到 4+,自动倒计时 3..2..1 → RunningView
8. 让模拟器走一会儿 (City Run 会自动行进,1 km 大约 12 分钟)
9. RunningView 应实时显示:
   - `实时配速` 6'00" 左右(City Run 是慢走)
   - 距离 0.20 km 渐增
   - 时长 1:34 渐增
   - **心率** 60-180 bpm 起伏(`#if targetEnvironment(simulator)` 的 sine 波 mock,真机才会有真心率)
10. 长按屏幕 0.6s → **PausedView**
11. 点"结束" 二次确认 → **PostRunView**
12. 看到真实距离/时长/平均配速
13. 点"+ 备注"(暂作"完成"用) → 回 IdleHome

## 7. 真机部署 (deferred 到 v0.5.x)

老 Mac (Xcode 12.5) 不支持 iOS 17,所以真机部署目前**无法验证**。所有验证停在模拟器。

真机带来的真实变化:
- HealthKit 心率: 模拟器读不到, 真机 + Apple Watch 配对才能拿真心率(否则 nil)
- 后台跑步: 需要 Background Modes Location updates 才不被 iOS suspend
- GPS 精度: 真机依赖位置和天气, 模拟器 City Run 总是"完美"
